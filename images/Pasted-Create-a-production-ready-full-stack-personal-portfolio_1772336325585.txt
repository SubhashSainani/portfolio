Create a production-ready full-stack personal portfolio website for a software developer.

Tech Stack:
Backend:
- Node.js (latest LTS)
- Express.js
- PostgreSQL
- Prisma ORM
- RESTful API
- Clean architecture
- Environment-based config

Frontend:
- React (latest version) with Vite
- Tailwind CSS for styling
- Framer Motion for animations
- Fully responsive (mobile + tablet + desktop)
- Modern, aesthetic, minimal, professional UI
- Smooth scrolling and subtle animations
- Clean typography and spacing
- Beautiful color palette suitable for a software engineer portfolio

IMPORTANT:
The design must look premium, modern, and recruiter-friendly.
It should feel like a professional SaaS landing page.
It must work perfectly on web and mobile.

--------------------------------------------------
PROJECT PURPOSE
--------------------------------------------------

This portfolio is meant to be shown to recruiters and hiring managers.
It should clearly communicate:
- Who I am
- My experience
- My projects
- My technical skills
- My certifications
- My education
- My soft skills
- Contact information

--------------------------------------------------
PAGES & SECTIONS
--------------------------------------------------

1. Home Page
- Hero section with:
  - Name
  - Title (e.g., Full Stack Developer)
  - Short professional summary
  - CTA buttons (View Projects, Contact Me)
- Professional profile image section
- Tech stack highlights
- Smooth scroll navigation

2. About Page
- Detailed professional summary
- Career goals
- Core strengths
- Soft skills (communication, leadership, teamwork, problem-solving)

3. Experience Page
- Timeline layout
- Job role
- Company name
- Duration
- Responsibilities
- Achievements with measurable impact (e.g., improved performance by 30%)
- Clean vertical timeline UI

4. Projects Page
- Modern card layout
- Project name
- Description
- Tech stack used
- Key features
- GitHub link
- Live demo link
- Filter by tech stack

5. Skills Page
- Technical Skills (categorized):
  - Frontend
  - Backend
  - Databases
  - DevOps
  - Tools
- Progress bars or modern skill badges
- Soft Skills section

6. Certifications Page
- Certification name
- Issuing organization
- Date
- Credential link
- Clean card/grid layout

7. Education Page
- Degree
- University name
- Year
- Relevant coursework
- Achievements

8. Contact Page
- Contact form (Name, Email, Message)
- Backend API to store messages in PostgreSQL
- Email validation
- LinkedIn, GitHub, Email links
- Download Resume button

--------------------------------------------------

Create APIs for:
- GET /api/profile
- GET /api/experience
- GET /api/projects
- GET /api/skills
- GET /api/certifications
- GET /api/education

Database tables:
- profile
- experience
- projects
- skills
- certifications
- education

Use:
- Proper validation
